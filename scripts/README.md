# Scripts

## compare_benchmarks.sh

Скрипт для сравнения производительности до и после изменений.

### Использование

```bash
# Первый запуск - создать baseline (уже сделано)
go test -bench=. -benchmem -benchtime=3s -run='^$' | tee benchmarks_baseline_before_adapters.txt

# После внедрения адаптеров - запустить сравнение
./scripts/compare_benchmarks.sh

# Или указать конкретный файл для сравнения
./scripts/compare_benchmarks.sh benchmarks_custom.txt
```

### Что проверяем

1. **Базовое логирование** - не должно замедлиться более чем на 5 ns/op
2. **Операции с контекстом** - не должны замедлиться более чем на 10 ns/op
3. **Аллокации памяти** - должны остаться без изменений
4. **Конвертация полей** - должна остаться на том же уровне

### Зависимости

Скрипт автоматически установит `benchstat` если он не установлен:
```bash
go install golang.org/x/perf/cmd/benchstat@latest
```

### Интерпретация результатов

`benchstat` показывает:
- `~` - нет существенной разницы
- `+X%` - производительность ухудшилась на X%
- `-X%` - производительность улучшилась на X%

Пример вывода:
```
name                              old time/op  new time/op  delta
Logger/zap-14                     0.35ns ± 0%  0.37ns ± 0%  +5.71%
Logger/xlog-14                    2.33ns ± 0%  2.98ns ± 0%  +27.90%
```
